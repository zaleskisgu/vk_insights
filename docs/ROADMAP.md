# План разработки

Опирается на [tz.md](./tz.md) и фактическое состояние в [IMPLEMENTATION.md](./IMPLEMENTATION.md).

## Уже есть

- Каркас Laravel + Vue: главная, форма сообщества и периода, `POST /report` с CSRF, валидация полей в контроллере.
- Мок `VkClient`, контракт и заготовка `HttpVkClient`; конфиг `config/vk.php`.
- Частичные UX-состояния на форме: загрузка и ошибка запроса.

## Приоритет 1 (ядро сдачи)

1. **`HttpVkClient`**
   - `groups.getById`, `wall.get` с `v` и `access_token` из конфига.
   - Пагинация `wall.get` до конца выборки или границы периода; фильтр постов по `from` / `to`.
   - Ошибки VK (`error` в JSON), 429 / сеть / таймауты — понятные ответы API; при 429 — retry/backoff по политике.
   - В `AppServiceProvider`: **`MockVkClient` | `HttpVkClient`** от `config('vk.use_mock')` и проверка токена при `use_mock = false`.

2. **Данные отчёта**
   - Проброс в сервис **реальных** `group` / период (парсинг screen name, ссылки → `group_id`, `owner_id`).
   - Агрегаты: топ по вовлечённости (лайки + комментарии + репосты), средняя вовлечённость, типы постов, динамика по дням.

3. **Кэш** (10–30 мин)
   - Ключ по группе и периоду; драйвер по выбору; TTL из конфига; промах → VK → расчёт → запись.

4. **REST и ТЗ**
   - Привести к `POST /analyze`, `GET /report/{groupId}?from=&to=` **или** зафиксировать в README осознанное отличие от ТЗ.
   - **`GET /health`**: алиас или отдельный маршрут рядом с `GET /up`.

5. **Наблюдаемость**
   - Лог: время обработки, число HTTP-вызовов к `api.vk.com`.

## Приоритет 2 (интеграция с фронтом)

6. Стабильная JSON-схема отчёта для дашборда.
7. Экспорт CSV и JSON.

## Приоритет 3 (фронт + оптимизация)

8. Дашборд: таблица + минимум один график; состояния **loading / empty / error** для блока результатов; адаптив.
9. **PERF.md** — 2–3 пункта «проблема → решение → эффект» + Lighthouse или Web Vitals до/после.
10. **README** — запуск, токен, мок/прод, опционально демо/GIF.

## Критерии из ТЗ (напоминание)

| Критерий      | Как закрывать |
| ------------- | ------------- |
| VK 35%        | Пагинация, 429, стабильность, мок для разработки |
| Backend 25%   | Кэш, явные маршруты, связка `VkClient` + отчёт |
| UX 20%        | Состояния, читаемый дашборд |
| Оптимизация 20% | `PERF.md` с цифрами |

При нехватке времени ТЗ допускает урезать объём в пользу **VK + кэш + базовый дашборд**.
